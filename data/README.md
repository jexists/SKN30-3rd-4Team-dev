# data/

오프라인 파이프라인 단계별 산출물을 저장합니다. 번호 순서대로 실행됩니다.

## 파이프라인 흐름

```
01_raw → 02_loaded → 03_processed → 04_chunks → 05_vectordb
```

## 폴더별 역할

| 폴더 | 단계 | 설명 | 형식 |
|------|------|------|------|
| `01_raw/` | 원본 수집 | 표준계약서 원본 파일 (수동 추가) | HWP, PDF 등 |
| `02_loaded/` | 문서 로드 | 원본에서 텍스트 추출한 체크포인트 | TXT, JSON |
| `03_processed/` | 전처리 | 정제·정규화된 텍스트 | JSON |
| `04_chunks/` | 청킹 | 임베딩 단위로 분할한 청크 체크포인트 | JSON |
| `05_vectordb/` | 벡터 저장 | Chroma 벡터스토어 | Chroma DB 파일 |

## 커밋 정책

- `01_raw` ~ `04_chunks`: 원본 및 중간 산출물 — 용량에 따라 판단
- `05_vectordb`: **의도적으로 커밋** (재임베딩 API 비용 절약)
  - 단일 파일 100 MB 초과 시 Git LFS 전환 검토
